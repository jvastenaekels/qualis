- [ ] **Step 4.2: Build `.gitleaksignore` for confirmed false positives**

`.gitleaksignore` uses fingerprints in the form `<commit-or-current>:<file>:<rule>:<line>`. Each line is one ignore.

For each false positive in the output, the ignore line is the value of the `Fingerprint` field from the JSON. Use jq to get it:

```bash
jq -r '.[] | .Fingerprint' "$RAW/gitleaks.json"
```

Example `.gitleaksignore`:

```
# Sample AWS credentials in published documentation (AKIAIOSFODNN7EXAMPLE).
# These are the official AWS sample values — not real credentials.
0e12257c1ee3f392e4c92aed52cd26257b0c8414:docs/guides/s3-setup.md:aws-access-token:71
0e12257c1ee3f392e4c92aed52cd26257b0c8414:docs/guides/s3-setup.md:aws-access-token:80
```

Use one line per false positive, with a comment block above each group explaining why it is safe to ignore.

## Task 7: Apply dependency CVE bumps (procedural — worked example)

For every finding from Task 6 that is classified `blocker` or `major` AND has a fix-version available AND requires only a one-line bump (no API breakage), apply this exact procedure. **One commit per dependency.**

**Files (per bump):**
- Modify: `backend/pyproject.toml` (Python deps) or `frontend/package.json` (Node deps)
- Modify: `backend/uv.lock` or `frontend/package-lock.json` (regenerated)

**Worked example: bumping `requests` from 2.32.5 to 2.33.0 (CVE-2026-25645)**

Use this exact pattern; substitute the package name and version for each remaining bump.

- [ ] **Step 7.1: Identify the bumpable list**

```bash
RAW=docs/audits/2026-05-03-comprehensive-security-audit/.raw/scanners/wave-1
jq -r '.dependencies[] | select(.vulns | length > 0) | select(.vulns[].fix_versions | length > 0) | "\(.name) \(.version) -> \(.vulns[].fix_versions[0]) (\(.vulns[].id))"' "$RAW/pip-audit.json"
```

This prints e.g. `requests 2.32.5 -> 2.33.0 (CVE-2026-25645)` for each fixable Python dep.

For npm:

```bash
jq -r '.vulnerabilities | to_entries[] | select(.value.fixAvailable != false) | "\(.key) -> \(.value.fixAvailable | if . == true then "auto" else .version end) (\(.value.severity))"' "$RAW/npm-audit.json"
```

```python
"""Regression tests pinned to CVEs fixed in Wave 1 of the 2026-05-03 audit.

For each fixed CVE, either:
  (a) write a test that exercises the formerly-vulnerable code path through
      Qualis's actual API, asserting safe behaviour, OR
  (b) skip with a docstring explaining why the CVE is not reachable from
      Qualis's call sites (so the test file documents the assessment).
"""
from __future__ import annotations

import pytest


def test_pyjwt_rejects_unknown_crit_header() -> None:
    """F-01-001 (CVE-2026-32597, pyjwt 2.12.0): unknown 'crit' extension must
    be rejected. Already pinned in pyproject.toml; this test guards against
    a future downgrade.
    """
    import base64
    import hmac
    import hashlib
    import json
    import jwt

    secret = "test-secret"
    header = {"alg": "HS256", "crit": ["x-custom-policy"], "x-custom-policy": "x"}
    payload = {"sub": "attacker"}

    def b64url(b: bytes) -> str:
        return base64.urlsafe_b64encode(b).rstrip(b"=").decode()

    h = b64url(json.dumps(header, separators=(",", ":")).encode())
    p = b64url(json.dumps(payload, separators=(",", ":")).encode())
    sig = b64url(
        hmac.new(secret.encode(), f"{h}.{p}".encode(), hashlib.sha256).digest()
    )
    token = f"{h}.{p}.{sig}"

    with pytest.raises(jwt.InvalidTokenError):
        jwt.decode(token, secret, algorithms=["HS256"])


@pytest.mark.skip(
    reason="CVE-2026-3219 (pip tar/zip confusion) only affects pip's install path. "
    "Qualis does not use pip at runtime — uv handles deploys. Tracked in backlog."
)
def test_pip_cve_2026_3219_not_reachable() -> None:
    pass
```
